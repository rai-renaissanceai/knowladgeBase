02 / Resumen

Resumen ejecutivo

Se revisó el rediseño del intake para Oferta Educativa, con foco en privacidad, homologación del proceso y manejo seguro de datos sensibles.

Se acordó separar bases operativas y privadas, restringir accesos y alojar la base privada en infraestructura interna o corporativa.

También se revisaron riesgos de grabaciones, uso de IA, validación psicométrica y planificación de Fase 0 / Fase 1.

Quedan pendientes la validación de formularios, definición final del alojamiento privado y programación de una demo del sistema automatizado.

03 / Decisiones y criterios

Decisiones tomadas y criterios definidos

Decisiones tomadas

  • Alojar la base de datos privada fuera de plataformas externas, en nube interna o servidores corporativos como Dropbox o SharePoint, con acceso limitado a líderes del proyecto.
  • Separar la base operativa de datos no sensibles de la base privada con información sensible.
  • Mantener la base privada sin conexión con el modelo de inteligencia artificial.
  • Eliminar grabaciones una vez extraída la plantilla de información, incorporando revisión humana previa para mitigar errores del modelo.
  • Avanzar con la unificación de procesos psicométricos como parte de Fase 1 para reducir redundancias.

Criterios definidos

  • La privacidad de datos debe mantenerse durante todo el proceso de intake.
  • Los consultores deben ver únicamente los casos bajo su responsabilidad.
  • El acceso debe restringirse mediante correos electrónicos.
  • Los datos sensibles deben clasificarse mediante semáforo o matriz de riesgo.
  • No toda la información recopilada debe almacenarse en plataformas del proyecto.
  • Los datos deben anonimizarse antes de cualquier procesamiento con inteligencia artificial.
  • Fase 0 se enfoca en tres pilares: intake, búsqueda y presentación de reportes.
  • La validación científica del sistema deberá apoyarse en pruebas estadísticas con grupo de control.

05 / Riesgos

Riesgos y bloqueos

  • Riesgo de exposición de datos sensibles si las grabaciones contienen información personal no filtrada.
  • Dependencia estratégica de plataformas externas para información sensible.
  • Riesgo de errores del modelo de IA durante la extracción automatizada; se definió revisión humana previa a eliminación de archivos.
  • Falta definición final del alojamiento seguro de la base privada.
  • Falta validar qué variables son esenciales para el proceso educativo y cuáles deben excluirse o anonimizarse.
06 / Backlog

Backlog y oportunidades futuras

  • Fase 1: creación de batería psicométrica completa que incluya competencias, habilidades y valores.
  • Fase 1: validación científica del sistema mediante pruebas estadísticas con empleados de Neoma como grupo de control.
  • Fase 1: diseño de dashboard para visualizar dimensiones y atributos del perfil del cliente.
  • Workflow futuro detectado: prerregistro de datos generales, sesión cualitativa, extracción automatizada, revisión humana y eliminación de grabación. Esto no va todavía. Lo dejamos en backlog.
